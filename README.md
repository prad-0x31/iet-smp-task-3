# CIPHER Deploy Hunt
### IET CIPHER · Summer Mentorship Program 2026 · Session 03 · **2-day task**

---

## The story

It's 9am. You just joined a startup as their first junior engineer. Your manager sends a Slack message:

> "The site is down. The last dev left the repo broken. Product launch is in 48 hours. Fix it."

This repo is broken in **5 different ways**. Deploy on Vercel, read the logs, fix each bug in order, get the app live.

The app UI is minimal on purpose: **`/`** is the broken app, **`/certificate`** is your reward when everything works.

---

## How to start

1. **Fork** this repo to your GitHub account.
2. **Connect to Vercel** — [vercel.com](https://vercel.com) → New Project → Import your fork.
3. **Deploy** and watch it fail.

---

## Rules

- Fix bugs **in order** (1 → 5).
- **Record each bug** as you go (format below).

---

## What to submit

For each stage:

```
Stage N
-------
Broken:  <paste the broken line(s)>
Fixed:   <paste the fixed line(s)>
Why:     <one sentence explaining what was wrong>
```
